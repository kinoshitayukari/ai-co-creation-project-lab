カリキュラム（ファイナンスデータサイエンス）1: AIエージェント × Python金融データ分析

金融データ分析読み物として再編集

「AIを使えば何かできそう」と思っても、いざ自分のことになると、何から始めればいいのか分からなくなることがあります。情報は多いのに、自分のテーマに落とし込もうとすると急に手が止まる。そんな不安は、とても自然なものです。

この記事では、学びを順番に並べ、迷わず進めるための講座設計として読めるように、元の設計メモを読み物として整理し直しました。

詳しい内容

コンセプト

ChatGPT Codex、Claude Code、CursorなどのAIコーディングエージェントを活用し、Pythonによる金融データ分析を「自分でコードを全部書く」のではなく「AIに指示して分析を実行させる」スタイルで学ぶカリキュラム。

従来のPython金融分析の教科書（『Pythonによるファイナンス 第2版』『Pythonからはじめるアルゴリズムトレード』）の体系的な知識を、AIエージェント時代のワークフローに再編成した独自構成。

注意事項

このカリキュラムは、投資助言や売買推奨を目的としたものではありません。金融データの分析手法・Python処理・AIエージェント活用の学習を目的とした教育コンテンツです。

第1部: 基盤構築（Week 1-2）

第1章: AIエージェントと金融データ分析の全体像

目的: AIエージェントで何ができるかを理解し、分析環境を整える。

内容:

  • AIコーディングエージェントとは何か（Codex / Claude Code / Cursor / Antigravity）
  • 従来のPython学習との違い: コードを書くからコードを指示するへ
  • エージェントに渡す「プロンプト」の基本構造
  • Google Colab / ローカル環境 / クラウドサンドボックスの選び方
  • 必須ライブラリ一覧: pandas, numpy, matplotlib, mplfinance, yfinance, requests

ワーク:

  • AIエージェントにpandas/numpy/matplotlibのインストールスクリプトを書かせる
  • Google Colabで動作確認する

第2章: Pythonの基礎をAIエージェントで学ぶ

目的: Python未経験でも、AIエージェントとの対話で基礎を身につける。

内容:

  • 変数、リスト、辞書、ループ、関数の基本
  • pandasのDataFrame操作（読み込み、フィルタ、集計）
  • numpyのベクトル演算
  • AIエージェントへの指示: 「このCSVを読み込んで、列Aの平均を出して」
  • エージェントの出力コードを読む力をつける

ワーク:

  • サンプルCSVをAIエージェントに渡して基本集計させる
  • 出力されたコードの各行の意味を確認する

第2部: 金融データの取得と整形（Week 3-4）

第3章: 金融データソースとAPI

目的: 分析に必要なデータをPythonで取得する方法を学ぶ。

内容:

  • yfinance / Yahoo Finance APIでの株価・為替・暗号資産データ取得
  • OANDA API / Binance APIの基本
  • Hyperliquid APIでの約定（fills）データ取得
  • FRED（米連邦準備制度）APIでの金利・経済指標取得
  • CSVファイルからのデータ読み込み

AIエージェント活用法:

プロンプト例:
「yfinanceを使って、BTCUSDの過去1年間の日足データを取得し、
CSVに保存するPythonスクリプトを書いてください。」

ワーク:

  • BTCUSD、日経225、米国10年国債利回りの3種類のデータを取得する
  • Hyperliquid APIから特定IDのfillsを取得する

第4章: データ前処理とクレンジング

目的: 生データを分析可能な状態に整える。

内容:

  • 欠損値の処理
  • タイムゾーン変換（UTC → JST）
  • OHLCV（始値・高値・安値・終値・出来高）データの構造理解
  • リサンプリング（M1 → H1 → Daily）
  • 複数データソースの結合（merge / join）

AIエージェント活用法:

プロンプト例:
「このCSVの日時列をUTCからJSTに変換し、
1時間足にリサンプリングしてください。欠損値はforward fillで補完。」

ワーク:

  • M1データをH1に変換するスクリプトをAIに書かせる
  • 金利データと価格データを日付で結合する

第3部: 金融分析の実践（Week 5-8）

第5章: データ可視化

目的: グラフとチャートで金融データを視覚的に理解する。

内容:

  • matplotlib / mplfinanceによるローソク足チャート
  • 移動平均線（SMA / EMA）のプロット
  • ボリュームの可視化
  • ヒートマップ（時間帯別・曜日別の変動率）
  • 相関マトリクス（複数資産の相関分析）

AIエージェント活用法:

プロンプト例:
「BTCUSDの日足ローソク足チャートに20日と50日の移動平均線を重ねて、
ボリュームも下部に表示してください。mplfinanceを使用。」

ワーク:

  • ローソク足 + 移動平均線チャートを作成する
  • 時間帯別変動率のヒートマップを作成する

第6章: 金融時系列分析

目的: 時系列データの統計的特性を理解する。

内容:

  • リターン計算（単純リターン / 対数リターン）
  • ボラティリティ（ヒストリカル / ローリング）
  • 正規性の検定（Shapiro-Wilk / Jarque-Bera）
  • 自己相関分析（ACF / PACF）
  • 定常性の検定（ADF検定）

AIエージェント活用法:

プロンプト例:
「BTCUSDの日次対数リターンを計算し、
正規性のShapiro-Wilk検定と、30日ローリングボラティリティを
グラフ付きで出力してください。」

ワーク:

  • BTCUSDのリターン分布を可視化する
  • ボラティリティの変動パターンを確認する

第7章: ポートフォリオ分析

目的: 複数資産の組み合わせを定量的に評価する。

内容:

  • シャープレシオの計算
  • 最大ドローダウンの算出
  • 効率的フロンティア（平均分散最適化）
  • リスク・リターンの散布図
  • 相関分析と分散効果

AIエージェント活用法:

プロンプト例:
「BTC、ETH、Gold、S&P500の過去2年間の日次リターンを使って、
等配分ポートフォリオのシャープレシオと最大ドローダウンを計算してください。」

ワーク:

  • 4資産ポートフォリオの効率的フロンティアを描く
  • 最適配分比率をAIに算出させる

第8章: バックテスト入門

目的: トレード戦略をPythonで検証する基本を学ぶ。

内容:

  • ベクトル化バックテストの仕組み
  • シグナル生成（ゴールデンクロス / デッドクロス）
  • 取引コスト・スリッページの反映
  • PF（プロフィットファクター）、勝率、最大DDの計算
  • ルックアヘッドバイアスの回避

AIエージェント活用法:

プロンプト例:
「BTCUSDの日足データで、20日SMAと50日SMAのゴールデンクロスで買い、
デッドクロスで売る戦略のバックテストを実装してください。
取引コスト0.1%を反映し、PF・勝率・最大DDを出力。
未来データの参照がないことを確認できるコードにしてください。」

ワーク:

  • SMAクロス戦略のバックテストを実行する
  • パラメータを変えてPFの変化を観察する

第4部: 機械学習と予測（Week 9-12）

第9章: 機械学習による市場分析

目的: scikit-learnを使い、価格データからパターンを検出する。

内容:

  • 特徴量エンジニアリング（テクニカル指標を特徴量に）
  • 分類モデル（ロジスティック回帰 / ランダムフォレスト / XGBoost）
  • 学習データとテストデータの分割（時系列対応）
  • ウォークフォワード検証
  • 過学習の検出と防止

AIエージェント活用法:

プロンプト例:
「BTCUSDの日足データから、RSI、MACD、ボリンジャーバンド幅を特徴量として、
翌日の上昇/下降をランダムフォレストで分類するモデルを作ってください。
時系列を考慮してウォークフォワード検証で評価。」

ワーク:

  • 3つの特徴量でモデルを学習・評価する
  • 精度が出ない場合の改善方法をAIに相談する

第10章: 深層学習の基礎（LSTM / Transformer）

目的: ディープラーニングによる時系列予測の概念を理解する。

内容:

  • LSTM（Long Short-Term Memory）の仕組み
  • 時系列データのシーケンス化
  • TensorFlow / PyTorchの基本
  • Transformerベースの時系列予測の概念
  • 予測精度の評価と限界の理解

AIエージェント活用法:

プロンプト例:
「BTCUSDの過去60日間の終値を入力として、
翌日の終値を予測するLSTMモデルをKerasで構築してください。
学習曲線と予測結果のグラフも出力。」

ワーク:

  • LSTMモデルを構築し、予測精度を確認する
  • 「予測モデルの限界」をAIに要約させる

第5部: 実践プロジェクト（Week 13-16）

第11章: Hyperliquidクジラ分析プロジェクト

目的: 実際のオンチェーンデータを使った総合分析プロジェクト。

内容:

  • Hyperliquid APIからのfills取得
  • 大口トレーダー（クジラ）のID抽出と分類
  • 取引パターンの可視化（エントリー時間、銘柄偏り、PF）
  • 複数クジラの同期トレード検出
  • BTCチャート上への取引ポイントプロット
  • 分析結果のレポート化

AIエージェント活用法:

プロンプト例:
「Hyperliquidのfills CSVから、closedPnlの合計が上位10のIDを抽出し、
各IDの取引頻度、勝率、PFを計算してください。
さらに、BTCのH1チャート上にそれぞれの取引ポイントを
三角マーカーでプロットしてください。」

第12章: 週次マーケットレポート自動生成

目的: 複数データソースを統合し、AIで自動レポートを生成する。

内容:

  • 金利、通貨、株価、コモディティ、暗号資産データの一括取得
  • データの自動集計とグラフ生成
  • AIによるマーケット概要の要約生成
  • Markdown / PDF形式でのレポート出力
  • 定期実行の仕組み（cron / GitHub Actions）

AIエージェント活用法:

プロンプト例:
「yfinanceとFRED APIから今週のBTC、S&P500、米国10年国債利回り、
金、原油のデータを取得し、前週比の変化率をテーブルにまとめ、
各資産のチャートを生成し、Markdownレポートとして出力してください。」

第13章: 受託・発信のためのポートフォリオ作成

目的: 学んだスキルを受託案件やコンテンツ発信に活かす。

内容:

  • 分析実績のデモレポート3本の作成
  • CrowdWorks / Lancers用の出品文作成
  • note記事・YouTube台本への変換
  • 投資助言にならない表現ガイドラインの適用
  • ポートフォリオサイトの構成

ワーク:

  • デモレポートをAIエージェントと共同で3本作成する
  • 受託サービスの価格表を整理する

付録B: プロンプト設計のコツ

金融データ分析をAIエージェントに依頼するときのルール:

  1. 入力データの形式を明記する（CSV、列名、期間）
  2. 出力形式を指定する（グラフ、テーブル、CSV、レポート）
  3. 使用ライブラリを指定する（pandas, mplfinance等）
  4. 投資助言ではないことを前提に含める
  5. ルックアヘッドバイアスなどの制約条件を明記する
  6. エラー時の対処方針を伝える

このカリキュラムの使い方

  1. 各章のワークをAIエージェントと一緒に実行する
  2. AIが出力したコードを必ず自分で読んで理解する
  3. 分析結果をスターターキットのレポートテンプレートに記入する
  4. note記事やYouTube台本に変換して発信する

重要: AIエージェントはコードを書く道具であり、投資判断を任せる道具ではありません。データ処理と可視化をAIに任せ、解釈と判断は自分で行ってください。
